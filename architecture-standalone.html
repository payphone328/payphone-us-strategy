Revenue

Revenue Model

How Payphone makes money on every $100 transaction cycle through the platform.

COBRAR Accept Payment

Customer Pays $100 via TAP

Stripe processes the NFC payment and takes its standard processing fee. Payphone charges a 1% platform fee on top.

$100.00 payment
- $3.20 Stripe fee (2.9% + $0.30)
- $1.00 Payphone platform fee (1%)
= $95.80 deposited to user's Financial Account
GASTAR Spend with Card

User Spends $95.80 with Visa Debit

When users spend with their Payphone Visa debit card, Payphone earns interchange revenue on every swipe/tap.

$95.80 card spend
x 1.5% interchange
= $1.44 interchange revenue to Payphone
CARGAR Cash Load

InComm Cash Load at Retail

Unbanked and cash-preferred users fund their account at 525K+ retail locations. Potential fee pass-through per cash load.

$3.95 per cash load at retail
= Additional revenue from unbanked / cash-preferred users

Total Revenue Per $100 Transaction Cycle

$2.44

$1.00 platform fee (COBRAR) + $1.44 interchange (GASTAR) = $2.44 per $100 throughput

+ additional cash load fees from CARGAR

Reference

Key Definitions

Platform Account
Payphone (master control — all users are sub-merchants under Payphone)
Custom Connected Account
Sub-merchant account per user (no Stripe dashboard — fully white-labeled)
Financial Account
User's fiat wallet (FDIC insured, ACH in/out, routing #, receives COBRAR payouts)
Issuing Card
User's Visa debit card (virtual + physical, funded by Financial Account balance)
PayFac
Payment Facilitator — Stripe's role as the registered entity enabling sub-merchant payment acceptance
BIN Sponsor
Fifth Third Bank / Evolve — the bank that provides the card BIN (Bank Identification Number) for Visa debit issuance
VanillaDirect
InComm's retail cash loading barcode system — generates unique barcodes users present at checkout to load cash
Bridge
Phase 2 cross-border off-ramp — converts USD to stablecoin to LATAM fiat for remittances

Every department's checklist for taking Payphone from architecture to launch in the U.S. market.

Engineering / Product

Build the Stripe-powered infrastructure and app experience

  • Set up Stripe Connect Platform Account
    Register Payphone as the Platform Account with capabilities: card_payments, transfers, treasury, card_issuing, us_bank_account_ach_payments. Configure platform-level settings and webhook endpoints.
    CriticalPhase 1
  • Build Custom Connected Account provisioning
    Auto-create a Custom Connected Account (sole proprietor sub-merchant) per user at sign-up. No Stripe dashboard exposed — fully white-labeled. Handle account verification lifecycle.
    CriticalPhase 1
  • Integrate Stripe Terminal SDK (COBRAR)
    Implement NFC TAP-to-pay using Stripe Terminal SDK. Phone becomes POS terminal. Handle card capture, payment creation, and real-time confirmation. iOS + Android.
    CriticalPhase 1
  • Implement Stripe Treasury (Financial Account)
    Auto-provision Financial Account per user with routing # + account #. FDIC pass-through via Fifth Third / Evolve. ACH in/out. This IS the Payphone wallet — display balance, transaction history, account details.
    CriticalPhase 1
  • Implement Stripe Issuing (GASTAR)
    Issue virtual Visa debit cards instantly. Provision to Apple Pay + Google Pay. Physical card ordering and fulfillment (5-7 day shipping). Real-time authorization, spend limits, card controls.
    CriticalPhase 1
  • Build ACH / debit funding flow (CARGAR)
    Stripe-powered on-ramp for bank account linking (via Plaid or manual entry). ACH pull/push. Debit card funding. Handle micro-deposit verification for manual bank link.
    HighPhase 1
  • Integrate InComm VanillaDirect (CARGAR)
    Generate unique barcodes per user for cash loading at 525K+ retail locations. Handle barcode display in-app, load notifications, and settlement reconciliation (InComm ACH to Financial Account).
    HighPhase 1
  • Integrate KYC / ID verification provider
    Select and integrate KYC provider (Socure, Veriff, Jumio, or Persona). Must support foreign government IDs (passports, consular IDs). Include liveness test and VPN/geolocation blocking.
    HighPhase 1
  • Build user onboarding flow (6-step journey)
    Download → KYC → Financial Account → Fund → Card → Start Using. Seamless mobile-first flow with progress indicators, error handling, and bilingual support (EN/ES).
    MediumPhase 1
  • Build transaction dashboard + notifications
    Real-time balance updates, transaction history for COBRAR / GASTAR / CARGAR. Push notifications for payments received, card charges, and cash loads. Export statements.
    MediumPhase 1
  • Build Bridge integration for ENVIAR
    Financial Account → Bridge → stablecoin → LATAM fiat. Cross-border remittance flow. Custom stablecoin via Bridge Open Issuance. 18-country card issuing for recipients.
    MediumPhase 2

Compliance / Legal

Regulatory framework, KYC ownership, and compliance infrastructure

  • Hire Chief Compliance Officer (CCO)
    Must have fintech / payments compliance experience. Responsible for overall compliance oversight, BSA/AML program, suspicious activity reporting, and regulatory liaison. Pedro Infante currently designated as CCO.
    CriticalPhase 1
  • Establish BSA/AML compliance program
    Written BSA/AML policies and procedures. Transaction monitoring rules. Suspicious Activity Report (SAR) filing procedures. Employee training program. Independent audit schedule.
    CriticalPhase 1
  • Select and contract KYC / ID verification provider
    Evaluate Socure, Veriff, Jumio, Persona. Key requirements: foreign government ID support (passports, consular IDs, matrícula consular), liveness detection, SSN/ITIN optional acceptance. Negotiate contract and SLA.
    CriticalPhase 1
  • Finalize Stripe Connect platform agreement
    Execute Stripe Connect Platform Agreement. Confirm PayFac model — Stripe holds PCI DSS Level 1, card network membership, and ACH originator status. Confirm Payphone has no MSB license obligation under this structure.
    CriticalPhase 1
  • Document foreign ID acceptance policy
    Formal policy for accepting non-US government IDs: which documents are accepted (foreign passports, consular IDs, national IDs), verification procedures, and edge cases. Align with Stripe's connected account verification requirements.
    HighPhase 1
  • Establish fraud monitoring + escalation procedures
    Define transaction velocity limits, unusual pattern detection, account takeover prevention. Escalation workflow from automated alerts to CCO review. Document cooperation protocols with Stripe's dispute management.
    HighPhase 1
  • VPN / geolocation blocking policy
    Select VPN detection and geolocation verification provider. Define policy: block account creation from VPN, prevent GPS spoofing during onboarding. Ongoing monitoring for suspicious location patterns.
    HighPhase 1
  • 1099-K tax filing coordination with Stripe
    Define process for users who provide SSN/ITIN. Stripe handles 1099-K filing for qualifying merchants. Document tax reporting responsibilities and data sharing between Payphone and Stripe.
    MediumPhase 1
  • Evaluate cross-border remittance licensing (ENVIAR)
    Determine if Payphone needs additional licenses for Phase 2 remittances via Bridge. Evaluate state-by-state MTL requirements for money transmission when sending funds cross-border.
    MediumPhase 2

Finance

Revenue model, unit economics, and financial operations

  • Finalize platform fee structure
    Confirm 1% platform fee on COBRAR transactions. Model competitive analysis against Square, Toast, SumUp. Ensure fee is sustainable at launch volumes and attractive to target merchant segment.
    CriticalPhase 1
  • Model interchange revenue share with Stripe
    Negotiate Payphone's share of 1.5% interchange on GASTAR (Visa debit spend). Understand Stripe Issuing's interchange revenue sharing model. Build projections based on average user spend.
    CriticalPhase 1
  • Negotiate InComm cash load fee structure
    Confirm $3.95 per cash load fee. Determine who absorbs the fee (user-paid vs. subsidized). Model cash load volume projections for unbanked / cash-preferred segment.
    HighPhase 1
  • Build unit economics model per user type
    Model revenue per Merchant, Regular, and Hybrid user. Factor in: platform fees, interchange, cash load fees, Stripe costs, KYC costs, card production, customer support. Target $2.44 per $100 throughput.
    HighPhase 1
  • Set up financial reconciliation for Stripe payouts
    Automate reconciliation of Stripe Connect payouts, Treasury balances, InComm ACH settlements, and interchange revenue. Daily/weekly/monthly financial reporting dashboards.
    HighPhase 1
  • Establish FDIC pass-through accounting
    Set up accounting treatment for user funds held in Stripe Financial Accounts (Fifth Third / Evolve). Ensure FDIC pass-through insurance compliance. Segregation of user funds from platform operating funds.
    MediumPhase 1
  • Model Phase 2 remittance revenue (ENVIAR)
    Project cross-border remittance fees. Model Bridge costs vs. traditional remittance pricing ($5-15 per transfer). Factor in 3.5% APY stablecoin yield as additional revenue stream.
    MediumPhase 2

Operations

Day-to-day operations, customer support, and operational integrations

  • Set up bilingual customer support (EN/ES)

    CriticalPhase 1
  • Establish dispute management workflow
    Define dispute handling process for COBRAR chargebacks and GASTAR card disputes. Coordinate with Stripe's built-in dispute management. Escalation paths, evidence submission, and resolution timelines.
    CriticalPhase 1
  • Physical card fulfillment logistics
    Set up physical Visa debit card ordering, printing, and shipping pipeline. 5-7 day delivery target. Card activation flow in-app. Handle replacements, lost/stolen reporting, and expedited shipping.
    HighPhase 1
  • InComm cash load operational procedures
    Document cash load flow: barcode generation → user presents at checkout → cash deposited → ACH settlement to Financial Account. Handle load failures, settlement delays, and reconciliation discrepancies.
    HighPhase 1
  • Build fraud monitoring + alerting operations
    Operational team for reviewing fraud alerts. Define velocity checks, unusual pattern detection, and account freeze procedures. Coordinate SAR filing with compliance. 24/7 monitoring for high-risk transactions.
    HighPhase 1
  • KYC review queue for manual verifications
    Handle edge cases where automated KYC fails: blurry IDs, unusual document types, failed liveness tests. Manual review queue with SLA. Escalation to compliance for suspicious applications.
    MediumPhase 1
  • Account closure and fund withdrawal procedures
    Define account closure process: withdraw remaining balance via ACH, cancel Visa card, deactivate Financial Account. Handle regulatory holds, dormant accounts, and unclaimed funds.
    MediumPhase 1
